# Coding Skills

Claude Code slash commands for Engineers. Scaffold RFCs, write tests, break down tickets, and draft postmortems.

<div class="grid cards" markdown>

-   :material-clipboard-list-outline:{ .lg .middle } **Ticket Breakdown**

    ---

    Break a JIRA ticket or feature description into detailed sub-tasks with acceptance criteria and implementation notes.

    **Command:** `/ticket-breakdown`

    [:octicons-arrow-right-24: View skill](DP-General-ticket-breakdown.md)

-   :material-test-tube:{ .lg .middle } **Test from Spec**

    ---

    Generate unit and integration tests from a feature spec or acceptance criteria.

    **Command:** `/test-from-spec`

    [:octicons-arrow-right-24: View skill](DP-General-test-from-spec.md)

-   :material-file-document-outline:{ .lg .middle } **RFC Scaffold**

    ---

    Generate a structured RFC document from a technical design proposal or ADR.

    **Command:** `/rfc-scaffold`

    [:octicons-arrow-right-24: View skill](DP-General-rfc-scaffold.md)

-   :material-alert-circle-outline:{ .lg .middle } **Postmortem Draft**

    ---

    Draft a blameless postmortem from an incident timeline and on-call notes.

    **Command:** `/postmortem-draft`

    [:octicons-arrow-right-24: View skill](DP-General-postmortem-draft.md)

-   :material-code-braces:{ .lg .middle } **Spec to Code Context**

    ---

    Convert a feature spec into a structured code context block for Claude to implement against.

    **Command:** `/spec-to-code-context`

    [:octicons-arrow-right-24: View skill](DP-General-spec-to-code-context.md)

</div>

---

!!! tip "How to install a skill"
    Copy the `.md` file into `~/.claude/commands/` (global) or `.claude/commands/` (project-local). Then run it with the slash command shown above.

# Coding Prompts

Single-purpose reusable prompts for Engineers. Copy, paste, and customize — no slash command needed.

<div class="grid cards" markdown>

-   :material-code-braces:{ .lg .middle } **Spec to Inline Comments**

    ---

    Convert a feature spec or PRD section into inline code comments — adds `// TODO`, `// NOTE`, and `// CONTEXT` annotations.

    [:octicons-arrow-right-24: View prompt](DP-General-spec-to-inline-comments.md)

-   :material-format-list-bulleted:{ .lg .middle } **Stories from Epic**

    ---

    Break a JIRA epic into individual, well-scoped stories with acceptance criteria and technical notes.

    [:octicons-arrow-right-24: View prompt](DP-General-stories-from-epic.md)

-   :material-source-pull:{ .lg .middle } **PR Description**

    ---

    Generate a structured pull request description from a diff summary or commit messages — ready to paste into GitHub.

    [:octicons-arrow-right-24: View prompt](DP-General-pr-description.md)

-   :material-file-document-outline:{ .lg .middle } **ADR Scaffold**

    ---

    Scaffold an Architecture Decision Record from a technical decision description. Covers context, options, decision, and consequences.

    [:octicons-arrow-right-24: View prompt](DP-General-adr-scaffold.md)

-   :material-wrench-outline:{ .lg .middle } **Tech Debt Ticket**

    ---

    Convert a code smell or architectural issue into a well-scoped JIRA tech debt ticket with impact, criteria, and effort estimate.

    [:octicons-arrow-right-24: View prompt](DP-General-tech-debt-ticket.md)

-   :material-alert-circle-outline:{ .lg .middle } **Incident Timeline**

    ---

    Construct a structured incident timeline from raw on-call notes, PagerDuty alerts, or Slack thread excerpts.

    [:octicons-arrow-right-24: View prompt](DP-General-incident-timeline.md)

</div>

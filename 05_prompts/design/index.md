# Design Prompts

Single-purpose reusable prompts for Designers. Copy, paste, and customize — no slash command needed.

<div class="grid cards" markdown>

-   :material-layers-outline:{ .lg .middle } **Component from Brief**

    ---

    Generate a Dialtone-conformant UI component design from a feature brief — including variants, states, and accessibility notes.

    [:octicons-arrow-right-24: View prompt](DP-General-component-from-brief.md)

-   :material-monitor-screenshot:{ .lg .middle } **UI from PRD**

    ---

    Generate UI screen flow suggestions from a PRD — layout, component choices, interaction patterns, and Dialtone mappings.

    [:octicons-arrow-right-24: View prompt](DP-General-ui-from-prd.md)

-   :material-wheelchair-accessibility:{ .lg .middle } **Accessibility Check**

    ---

    Check a design against WCAG 2.1 AA and Dialpad accessibility standards. Outputs a severity-rated issue list with Dialtone-specific remediation.

    [:octicons-arrow-right-24: View prompt](DP-General-accessibility-check.md)

-   :material-file-export-outline:{ .lg .middle } **Handoff Notes**

    ---

    Generate structured engineering handoff notes from annotated Design Studio screens — Dialtone component names, props, aria attributes, and interaction behavior.

    [:octicons-arrow-right-24: View prompt](DP-General-handoff-notes.md)

-   :material-comment-check-outline:{ .lg .middle } **Design Review Summary**

    ---

    Summarize a Gate review or Design Studio critique session into a structured summary with decisions, required changes, and next steps.

    [:octicons-arrow-right-24: View prompt](DP-General-design-review-summary.md)

</div>

# Recommended AI Tools for Product Managers at Dialpad

## Overview

This guide lists the AI tools PMs at Dialpad use across the product development lifecycle —
from discovery through launch. Tools are organized by workflow stage. Each entry includes
the recommended use case, how it connects to the AI Playbook, and any Dialpad-specific
configuration notes.

**Daily baseline:** Most PMs default to **Glean + Gemini** for everyday work.
Claude Code layers on top for structured workflows (PRDs, JIRA, GTM).

---

## Primary Tools by Workflow

### 1. Glean
**Best for:** Finding internal Dialpad context — past PRDs, Confluence docs, customer research,
competitive intel, prior roadmap decisions. Also your cross-app work assistant (Gmail, Drive,
Jira, Confluence, Salesforce, Zendesk, Monday, GitHub).

Use Glean before drafting a PRD to:
- Find existing research on the feature area
- Check whether a similar feature was scoped before
- Pull accurate product names, pricing tiers, and customer segment data

**Glean MCP:** If the Glean MCP is configured in Claude Code (`09_mcps/`), you can run Glean
searches directly inside agent sessions without switching tools.

---

### 2. Gemini
**Best for:** Everyday writing assistance — email drafts, meeting summaries, doc editing,
spreadsheet formulas, slide generation, quick analysis. Gemini is built into Gmail, Docs,
Sheets, Slides, and Drive so no context switching is needed.

Use **Gemini Canvas** for collaborative drafting of scripts, briefs, and lightweight tools
where you want a co-editing workspace rather than a single-turn output.

Not recommended for gate-quality PRDs or Dialpad-specific artifacts — use Claude Code + this
repo for those.

---

### 3. Claude Code (claude.ai/code)
**Best for:** PRD drafting, task breakdown, GTM artifact generation, gate review prep

Claude Code runs the skills in `03_skills/` and agents in `04_agents/` as slash commands.
It is the right tool when you need Dialpad-context-grounded, structured artifacts.

| Workflow | Skill / Agent |
|---|---|
| Draft a PRD from a brief | `/draft-prd` |
| Expand a weak PRD section | Use `05_prompts/pm/DP-General-prd-section-expander.md` |
| Break PRD into JIRA epic | `/generate-jira-epic` |
| Generate release notes from a JIRA ticket | `/release-notes-from-ticket` |
| Prep for Gate 1 review | `/gate-review-checklist` |
| Draft GTM messaging | `/gtm-draft` |
| Full PRD → JIRA pipeline | `04_agents/pm/DP-General-prd-generator/` |

**Setup:** Install Claude Code CLI, open the `DPAIPlaybook` folder as your working directory.
The `CLAUDE.md` in the repo root loads automatically.

---

### 4. Atlassian Rovo
**Best for:** AI assistance inside Jira and Confluence — Q&A on existing docs, ticket creation
and updates, workflow automations without leaving Atlassian tools.

Use Rovo when you need to work within an existing Jira epic or Confluence doc rather than
generate a net-new artifact. For gate-quality structured output, use Claude Code + this repo.

---

### 5. Design Studio
**Best for:** AI-powered UI prototyping inside Design Studio for Gate 1 review

Use Design Studio to generate UI screens from a text prompt directly in Design Studio, then remap to
Dialtone components. The prompt in `05_prompts/design/DP-General-ui-from-prd.md` is
designed to produce a Dialpad-aligned generation prompt.

vs. a full Designer-led Design Studio build.

---

### 6. monday.com AI
**Best for:** AI agents and sidekick assistance within the monday.com project management
platform — status updates, task generation, and sprint planning from within Monday boards.

---

### 7. JIRA MCP (via Claude Code)
**Best for:** Creating and syncing tickets from story output generated by Claude Code

Claude Code can generate JIRA-ready epic and story descriptions. With the JIRA MCP configured
(see `09_mcps/guides/DP-General-atlassian-setup.md`), tickets are created directly from the
agent session without copy-paste.

## Tool Selection Guide

| I need to... | Use this |
|---|---|
| Draft a PRD from a brief | Claude Code + `/draft-prd` |
| Find past research or decisions | Glean |
| Summarize a meeting or draft an email | Gemini (in Gmail / Docs) |
| Draft a doc collaboratively | Gemini Canvas |
| Create a UI prototype | Design Studio |
| Generate JIRA tickets from PRD | Claude Code + `/generate-jira-epic` + JIRA MCP |
| Update a Confluence doc or Jira ticket | Atlassian Rovo (in-app) |
| Draft What's New entry from a JIRA ticket | Claude Code + `/release-notes-from-ticket` |
| Draft full GTM messaging from PRD | Claude Code + `/gtm-draft` |
| Check Gate 1 readiness | Claude Code + `/gate-review-checklist` |
| Manage sprint tasks in Monday | monday.com AI |

---

## Not Recommended

- **ChatGPT for gate-quality PRDs** — limited seats; no Dialpad context; outputs require heavy manual correction. Use Claude Code + this repo instead.
- **Copilot for PM work** — optimized for code, not product artifacts
- **AI tools without Dialpad context loaded** — outputs will use generic personas and miss product-specific constraints

---

## Getting Started

1. Install Claude Code: `npm install -g @anthropic-ai/claude-code`
2. Open `DPAIPlaybook` as your working directory
3. Run `/draft-prd` to see your first skill in action
4. See `00_getting-started/` for a step-by-step onboarding guide

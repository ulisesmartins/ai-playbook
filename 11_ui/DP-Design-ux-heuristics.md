# UX Heuristics & Scorecards

Heuristic evaluations are a method for discovering potential usability problems within user interfaces and evaluating compliance with industry-standard usability principles.

## Categories

| Value | Heuristic |
|-------|-----------|
| **Simple and obvious** — Achieve the same outcome in fewer steps, or a better outcome in the same steps | [Match between system and the real world](https://www.nngroup.com/articles/match-system-real-world/) — Experiences should speak the users' language using familiar words, phrases, and concepts rather than system-oriented terms. |
| | [Aesthetics and minimalistic design](https://www.nngroup.com/articles/aesthetic-minimalist-design/) — Interfaces should not contain irrelevant information. Every extra unit of info competes with the relevant units. |
| **Effortless** — Help the user get to the expected outcome as quickly as possible | [Consistency and standards](https://www.nngroup.com/articles/consistency-and-standards/) — Users should not have to wonder whether different words, situations, or actions mean the same thing. |
| | [Error prevention](https://www.nngroup.com/videos/usability-heuristic-error-prevention/) — Good error messages are important, but the best designs prevent problems from occurring in the first place. |
| **Scalable** — Make sure the workflow works at scale | [Flexibility and efficiency of use](https://www.nngroup.com/articles/flexibility-efficiency-heuristic/) — Shortcuts may speed up interaction for expert users. |
| | [Recognize, diagnose, recover from errors](https://www.nngroup.com/videos/usability-heuristic-recognize-errors/) — Error messages should be in plain language, precisely indicating the problem and suggesting solutions. |
| **Discoverable** — The user can easily discover the feature | [Recognition rather than recall](https://www.nngroup.com/articles/recognition-and-recall/) — Minimize user memory load by making elements, actions, and options visible. |
| | [Visibility of system status](https://www.nngroup.com/articles/visibility-system-status/) — Designs continually inform users on their status through appropriate and timely feedback. |
| | [Help and documentation](https://www.nngroup.com/articles/help-and-documentation/) — When necessary, users are given help to complete their tasks. |
| | [User control and freedom](https://www.nngroup.com/articles/user-control-and-freedom/) — Allow users to exit a flow or undo an action and go back to previous system state. |

We also track: **Time on task** and **Number of clicks** (if applicable).

## Scoring

| Grade | Score | Summary | Ease | Experience |
|-------|-------|---------|------|------------|
| **GRADE A** | 5 | Exceeds expectations — Heuristic is met in every way and optimized. Users may report the task is delightful. | Extremely easy | Extremely good |
| **GRADE B** | 4 | Somewhat exceeds expectations — Heuristic is met almost in every way with no more than minor exceptions. | Successful | Good |
| **GRADE C** | 3 | Meets expectations — Heuristic is mostly met. Minor workarounds may be needed. | Okay | Okay |
| **GRADE D** | 2 | Falls short of expectations — Heuristic is lacking in significant ways leading to user frustration. | Difficult | Bad |
| **GRADE F** | 1 | Does not meet expectations — Heuristic isn't met. Task may be impossible to complete. | Very difficult | Poor |

## Why Heuristic Evaluations?

### For Product Designers

- **Identifying Usability Issues** — Uncover challenges within the user experience workflow
- **Early Problem Detection** — Catch issues during the design phase, saving time and resources
- **Objective Evaluation** — Based on established heuristics, providing objective assessment
- **Iterative UI Improvement** — Feedback enables iterating and refining workflows
- **Enhanced User Satisfaction** — Well-designed workflows lead to improved satisfaction

### For Product UI Leadership

- **Ensuring consistency** across design guidelines and standards
- **Validating design decisions** and gaining alignment insights
- **Improving overall product quality**

### For PMs, ENG, and Stakeholders

- **Aligning with Industry Best Practices**
- **Saving Time and Resources** through early issue identification
- **Improving Collaboration** across cross-functional teams
- **Enhancing Brand Reputation** through quality user experiences

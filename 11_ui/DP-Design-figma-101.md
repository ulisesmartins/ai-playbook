# Using Figma at Dialpad 101

Figma is our main tool for designing product experiences at Dialpad. This section gives you everything you need to get started — from starter files to the libraries that help keep things consistent across projects.

Using these shared libraries is required — it ensures that everyone's working from the same structure and speaking the same visual language. That said, don't hold back — bring your creativity into every project.

## Primary UI Tool

Our primary design tool is [Figma](https://www.figma.com/). As a Product Designer or Product UI Manager, you will have access to Figma and the Dialpad team projects. You can access Figma via Okta.

Anyone else in Dialpad can access your files when you share a file URL or invite them via email. We ask that you only grant non-Designers "view" permissions. Anyone given "edit" access is a paid seat and must be approved by their department head. A user with "view" permissions can comment on and inspect design files.

## Access & Permissions

Figma has four access levels, also called Permissions. Permissions can be set at a team, project, and file level.

| Role | Description |
|------|-------------|
| **Editors** | Only Product Designers and Product UI Managers in the UX department are editors. Editors have "can edit" access to projects and files unless otherwise changed. |
| **Viewers** | Has "view" access to any file within a project. If invited, viewers are able to be granted editor permission. This is a paid seat and should only be done with a department head's written approval. |
| **Viewer-Restricted** | A viewer role, but locked and can only be upgraded to an editor role if promoted by an admin. |
| **Admins** | Has the ability to manage users, teams, and projects. |
| **Owners** | Editors or admins who create a team, project, or file will be the default owner. |

## Dialtone Figma Libraries

These libraries are part of Dialpad's design system, [Dialtone](https://dialtone.dialpad.com), and are essential for ensuring consistency across brand, user experience, and design in every project.

| Library | Description | Link |
|---------|-------------|------|
| **DT9 Component Library** | Core UI building blocks used across Dialpad products. | [View in Figma](https://www.figma.com/design/W58r5BkO8qTw3vem9YieJd/DT9-Component-Library--Rebrand-2025-) |
| **DT9 UI Tokens** | Core styles like colors, spacing, and typography for theming. | [Dialtone docs](https://dialtone.dialpad.com/tokens/) |
| **DT9 Icon Library** | Collection of standardized icons for consistent visual language. | [View in Figma](https://www.figma.com/design/zz40wi0uW9MvaJ5RuhcRZR/DT9-Icon-Library) |
| **Spot Illustrations** | Branded artwork used to enhance empty states and visuals. | [Dialtone docs](https://dialtone.dialpad.com/design/illustrations/) |
| **App Components** | App-specific components tailored to Dialpad's products. | [View in Figma](https://www.figma.com/design/oTY56LgQha8ljb4c8w4GtO/Dialpad-App-Components) |
| **DT9 UI Tokens (Dark Mode)** | Dark mode variants of core design tokens. | [Dialtone docs](https://dialtone.dialpad.com/tokens/?theme=dark) |
